# 《末班地铁》设计文档
> 丧尸末日生存 · 文字冒险游戏 | 手机浏览器运行 | 设计定稿 2026-07-04
> 本文档是跨会话扩写剧情的唯一权威依据。任何新剧情必须与本文档的设定、大纲、数值、命名规范一致。

---

## 1. 项目概述

- **游戏名**:《末班地铁》
- **类型**:文字冒险(分支剧情 + 轻度数值)
- **平台**:手机浏览器优先(竖屏),桌面浏览器兼容
- **部署**:纯静态文件 → GitHub Pages;另提供单文件打包 `offline.html`
- **文本量目标**:≥50万字,分阶段累积(见 §9)
- **结局数**:28 个(18 命运大结局 + 10 叙事性死亡结局)
- **基调**:写实压抑 + 人性拷问;丧尸是背景,人才是考题。主角的社畜特质(服从、忍耐、麻木)既是软肋也是武器。内心吐槽带黑色幽默,克制使用。

## 2. 世界观与病毒设定

- **舞台**:深圳,真实地理。核心地点:南山科技园(公司"辰星网络"所在写字楼22层)、白石洲城中村(主角合租屋)、深南大道、市人民医院(罗湖)、蛇口港、梧桐山军方临时安置区、昆仑生物大厦(科技园)。
- **爆发之夜**:台风"海鳗"登陆深圳的周五凌晨(游戏内 Day 0)。暴雨、部分停电、信号时断时续,为初期信息迷雾提供合理性。
- **病毒「鸿茅药酒」(实验室编号 HM-7)**:科技园企业"昆仑生物"研发的口服神经兴奋类药剂,以药酒形态包装上市前测试,内部代号"鸿茅药酒",立项卖点为"抗疲劳效率增强"(让人不困、不累、持续专注——终极加班药)。临床失败后转入秘密人体试验;台风夜实验区断电,试验体失控外逸。(注:与真实品牌同名系用户指定;若公开发布建议改为戏仿名以规避商标/名誉纠纷。)**主题讽刺:丧尸就是终极社畜——不眠不休,永不停止"工作"。**
- **感染规则**(全线一致,不得违背):
  - 咬伤/体液入创口→高概率感染;潜伏期6~48小时;前兆:低烧、瞳孔泛灰、闻到腐味觉得香。
  - 丧尸:趋声趋光,雨天嗅觉失灵(台风雨=天然掩护,雨停=难度上升),奔跑速度=生前体能(西装丧尸慢,骑手丧尸快)。
  - **陈默的秘密**:两年前为还花呗参加过昆仑生物的有偿药物试验(受试者编号 HM-041),体内有部分抗性——被咬不一定转化(中期揭示,真相线核心钩子,「第一批疫苗」结局的依据)。此设定在序章埋暗线(体检报告、手臂旧针疤),不得提前明示。

## 3. 主角与NPC

- **陈默**,男,25岁,辰星网络后端程序员。月薪1万8,白石洲合租,入职三年,手机存着没敢发的辞职信。父亲早逝,母亲在湖南老家。性格:隐忍、观察力强、习惯性服从但内心毒舌。**叙事人称:第二人称"你"**,他人称呼他"陈默"。
- **阿凯**(郑凯),26,室友,外卖骑手。熟悉全城路况,有电动车,乐观仗义。固守/撤离线核心同伴。好感变量 `aff_akai`。
- **程霜**,24,女友,市人民医院护士。恋爱三年,同城却像异地(他加班她倒班)。Day 0 凌晨发来最后短信:"医院来了好多奇怪的病人,你别出门。" 寻人线目标。`aff_cheng`。
- **老周**(周国栋),52,写字楼保安,退伍侦察兵。序章导师型角色,教玩家生存规则。`aff_zhou`。
- **许乐**,23,应届生同事,技术宅,会修无人机,胆小话痨。真相线关键帮手。`aff_xule`。
- **康明远**,45,辰星网络CEO。序章带私人保镖抢先撤离;后期在安置区以"重建委员会"名义再现,权力型反派。
- **小满**,8岁,白石洲便利店老板的女儿,爆发夜父母双亡。道德试金石,多条线可遇。`aff_xiaoman`。
- **陈母**,48,湖南老家。电话/短信亲情线,撤离线动机。
- **钢牙**(原名不详),35,掠夺者团伙"钢牙帮"头目,原龙华包工头。堕落线核心。
- **秦鹭**,31,昆仑生物研究员,HM-7「鸿茅药酒」项目组幸存者,掌握真相与实验室门禁。真相线关键NPC。`aff_qin`。
- **郑上校**,梧桐山安置区指挥官。秩序至上,非脸谱化恶人——他的每个冷酷决定都有军事逻辑上的合理性。
- 群像:白石洲房东嬢嬢、五金店老蔡、网吧少年等,按需在固守线扩充。

## 4. 数值与条件系统

| 变量 | 键名 | 范围/初始 | 说明 |
|---|---|---|---|
| 体魄 | `hp` | 0~100,初始60 | 社畜身体虚。归零→就地死亡结局。影响体力型选项 |
| 理智 | `san` | 0~100,初始70 | <30 解锁疯狂选项/锁定冷静选项/文字发疯渲染;归零→结局 D10 |
| 人性 | `humanity` | -100~+100,初始0,**玩家不可见** | 道德选择累积。≤-30 可入堕落线;多结局判定用 |
| NPC好感 | `aff_*` | 0~10 | 见 §3。决定救援/同行/牺牲桥段 |
| 物品 | `items` | 集合 | 关键道具:消防斧 fireAxe、员工门禁卡 keycard、昆仑门禁卡 kunlunCard、抗生素 antibiotics、车钥匙 carKey、程霜的字条 note_cheng、收音机 radio 等 |
| 剧情标记 | `flags` | 布尔集合 | 线索、事件、承诺。真相线索统一前缀 `clue_` |
| 游戏时间 | `day` | Day 0 起 | 顶部状态栏显示 |

选项条件 `require` 支持:`hpAbove/hpBelow、sanAbove/sanBelow、humanityAbove/humanityBelow、item、noItem、flag、noFlag、aff:{npc,gte}`。不满足→灰色锁定并显示缺失原因(可配置隐藏)。

## 5. 选项颜色与文字动效

**颜色语义(CSS class = 颜色名)**:
- `white` 中性推进|`red` 危险/激进(微颤动画)|`green` 谨慎/稳妥|`blue` 理性/调查(常发线索)|`purple` 条件解锁(微光浮现,标注来源)|`gold` 命运抉择(呼吸光晕+屏幕暗化聚焦,选前提示"此选择将改变你的命运")
- 灰色锁定态:`locked`,显示 🔒 与缺失条件。

**文字动效**:打字机逐字(可调速/关闭/点击跳过);低理智(san<30)正文随机抖动、闪烁、插入血红一闪而过的幻觉短句;死亡结局血色渐染+文字碎裂;数值变化飘字(+/-);物品得失、NPC首次登场专属高亮色。所有动效尊重 `prefers-reduced-motion`。

## 7. 剧情结构与各线大纲

**幕结构**:序章 ch0 → 第一幕 ch1(枢纽)→ 第二幕六线 → 第三幕安置区交汇 ch3 → 终幕结局 endings。

- **ch0 序章·写字楼**(Day 0 凌晨,线性+小分支):22层加班→爆发→逃生(楼梯/电梯/天台三路)→遇老周→出楼。教学:颜色语义、数值、物品(消防斧/门禁卡可得)。埋线:昆仑生物同楼层快递箱、陈默针疤。出口死亡结局:D01、D02。
- **ch1 第一幕·雨夜穿城/白石洲**(Day 0~2):序章已完成科技园→白石洲的穿城,本幕承接白石洲楼下的阿凯 SOS。事件池:七层握手楼攀爬(逐层邻里遭遇,呼应序章22层下楼但基调转为社区/人情)、天台救阿凯(D04坠雨风险)、低理智时的暗号幻听(D05,承接阿凯 SOS 暗号)、救人抉择(D03:救了伪装伤员的劫匪)、房东嬢嬢与五金店老蔡等城中村群像登场、小满(saw_child 回收)、老周电台433整点呼叫窗口(radio/zhou_channel 回收)。白石洲确立为枢纽。本幕末尾 gold抉择决定第二幕入口(守/寻/走/海)。真相线、堕落线为隐藏入口(解锁条件见本节各线条目)。节点前缀 `ch1_`。
- **ch2-hold 固守线**:白石洲改造(物资、水、围栏)、收留难民的道德题、房东与租客派系内斗、钢牙帮上门"收租"、大规模尸潮攻楼。→E01/E02/E03。
- **ch2-rescue 寻人线**:横穿罗湖(桥/隧道/地铁隧道徒步),医院沦陷区搜索,发现医院地下秘密(D07,兼真相线索),程霜的三种命运(未感染/已感染/已离开留字条)由到达速度与flag决定。→E04/E05/E06。
- **ch2-north 撤离线**:与阿凯组车队北上,梅观高速关卡、东莞收费站黑市、车队内部信任危机、带孩子的同伴之死。→E07/E08/E09。
- **ch2-sea 海路线**:蛇口港,船票黑市(金条?抗生素?还是人),海关缉私艇争夺,同舟者群像。→E10/E11/E12。
- **ch2-truth 真相线**(隐藏,需 `clue_*`≥2 + kunlunCard 或 aff_qin≥3):昆仑生物大厦潜入,HM-7「鸿茅药酒」档案,发现自己是 HM-041,秦鹭的抉择,证据的用途三分。→E13/E14/E15。
- **ch2-fall 堕落线**(隐藏,humanity≤-30 且遇钢牙帮):投名状(D09)、帮派生活的糖与血、小满再现良心拷问。→E16/E17,可跳E03。
- **ch3 安置区交汇章**:多线可汇入梧桐山安置区(固守失败/撤离受阻/寻人后等)。配给制、身份等级、康明远的"重建委员会"、郑上校的困境、哗变之夜(D08)。E18 与部分线尾声在此触发。

**跨线一致性规则**:同一 Day 的全城大事件(全城停电 Day 2 夜、军方宣布梧桐山安置区 Day 4、暴雨停歇 Day 6=丧尸嗅觉恢复)在所有线中一致出现。

## 8. 技术架构

```
文字游戏/
├── index.html          入口:标题屏(新游戏/继续/存档/图鉴/设置)
├── css/style.css       暗黑主题、6色选项、动效、竖屏优先
├── js/engine.js        节点渲染、条件判定、效果结算、路由
├── js/typewriter.js    打字机与文字特效
├── js/save.js          存档系统
├── js/gallery.js       结局图鉴(28格,未解锁=剪影+模糊提示语)
├── js/story-index.js   章节注册表
├── story/ch0-prologue.js / ch1-crossing.js / ch2-*.js / ch3-camp.js / endings.js
├── tools/build.js      打包单文件 offline.html(Node,可选)
└── docs/superpowers/specs/  设计文档(本文件)
```

**节点格式**(剧情文件 = `window.STORY.register(id, nodes)`):
```js
"h2_012": {
  text: "……",                    // 支持少量内联标记:{item:消防斧} {npc:老周} {red:一闪而过的幻觉}
  effects: { san: -5, hp: 0, humanity: -10, item: "+fireAxe", aff_akai: 1, flag: "clue_lab" },
  choices: [
    { text: "…", color: "red|green|blue|purple|gold|white",
      require: {…}, effects: {…}, goto: "节点id" }
  ],
  ending: "E01"                   // 结局节点专用,登记图鉴
}
```

**节点ID规范**:`ch0_/ch1_/h2_(hold)/r2_(rescue)/n2_(north)/s2_(sea)/t2_(truth)/f2_(fall)/c3_(camp)` + 三位序号;死亡结局节点 `die_D01` 等;结局节点 `end_E01` 等。

**存档**:localStorage;自动存档(每节点)+ **5个手动存档槽**,槽位显示章节名/游戏内Day/现实日期/数值快照,支持覆盖、删除;导出/导入 Base64 存档码(跨设备)。图鉴解锁独立于存档持久保存。章节回溯:到达过的章节起点可直接开新周目。

**移动端**:竖屏单栏,正文≥17px,选项全宽按钮点击区≥44px,顶部一行状态图标;默认深色主题;禁双击缩放;`prefers-reduced-motion` 降级动效。

**写作规范(每次扩写必读)**:第二人称"你";句子短促,细节具体(品牌、气味、声音),煽情克制;黑色幽默只出现在陈默的内心吐槽;单节点正文100~300字,重场面≤500字;每个选择的后果必须在3个节点内产生可感知差异(文本或数值);gold抉择每章≤2个。
